# Q1: Core Functional Modules of the Business Logic Layer

For the **Nexus Intelligent Chatbot System**, the Business Logic Layer (BLL) orchestrates the core rules, processing natural language commands, handling authentication, and interacting with various external systems. 

## Identified Core Functional Modules

1. **Authentication & Identity Module (`auth/`)**
   - **Responsibility:** Handles user login, registration, role management (General User vs. Administrator), and token generation/validation (JWT).
   - **Business Rules:** Validates user credentials against the database, issues scoped tokens, and enforces access control on sensitive actions (e.g., executing scripts).

2. **NLP Engine & Parsing Module (`model/`)**
   - **Responsibility:** Processes incoming natural language prompts from the user, extracts intents and entities, and determines the confidence of the parsed command.
   - **Business Rules:** If the confidence score is below a predefined threshold (e.g., < 50%), it triggers a fallback response instead of executing an incorrect task.

3. **Chat & Execution Orchestration Module (`chat/`)**
   - **Responsibility:** Acts as the central hub connecting the NLP output to actionable tasks. It manages contextual memory to provide conversational continuity.
   - **Business Rules:** Queries system metrics, invokes the script engine, and ensures sensitive output is masked before returning responses to the user.

## Interaction with the Presentation Layer (User Interface)

The Presentation Layer, housed in the `client/` directory and built with React (`App.jsx`, `main.jsx`), communicates with the BLL via REST API calls. 

**Interaction Flow:**
1. **User Input:** A user types a message or command in the React UI.
2. **API Request (Presentation to BLL):** The frontend sends an HTTP POST request to the Chat API (`chat/chat_api.py`), including the user's prompt and authentication token.
3. **BLL Processing:**
   - The token is passed to the Authentication Module to verify identity and permissions.
   - The prompt is routed to the NLP Engine (`model/main.py`) to parse intent and extract entities.
   - Based on the intent, the Execution Orchestration layer triggers the appropriate script or fetches the required data.
4. **API Response (BLL to Presentation):** The BLL packages the execution result or conversational response (often masking sensitive info if required) into a JSON payload and returns it to the client.
5. **UI Update:** The React application receives the JSON data, updates its state to display the chatbot's response, and renders any system metrics or alerts dynamically inside the chat interface window.
